# Why Run a Node? - Meetup #4 Summary
**Date:** September 8, 2022

## Community Mission & Rules
- Mission: Share bitcoin knowledge, tools, resources to grow adoption in Round Rock
- Rules: No shitcoining, no pictures/video without agreement, respect establishment/families
- Socials: Twitter @rbitcoiners, Matrix Element

## Types of Bitcoin Nodes
| Node Type | Primary Function |
| Altruistic Node | Validates blockchain; strengthens decentralization; asserts rules |
| Economic Full Node | Verifies own transactions; connected to personal wallet |
| Mining Node | Participates in Nakamoto Consensus; earns rewards/fees |
| Lightning Node | Layer 2 transactions; Connected Wallet or Routing Node |
| "Uncle Jim" Node | Privacy-focused; verifies for family/friends; CoinJoins |

## Popular Node Projects
- Bitcoin Core: reference implementation; ~500GB space
- RaspiBlitz: DIY, fully open-source, Raspberry Pi
- Umbrel / MyNode / Nodl: plug and play, Lightning enabled
- Start9 Embassy: self-sovereign computing device
- RoninDojo: privacy-focused, on-chain only, integrates Whirlpool and Bisq

## Hardware Wallets
- Trezor: open-source, non-custodial
- Ledger: closed-source, Secure Element
- ColdCard: verifiable-source, air-gapped
- Seed Signer: DIY, fully open-source

## Connecting Wallets to Nodes
1. HWI with Bitcoin Core (CLI, advanced)
2. Electrum Server (Umbrel/MyNode, Tor, remote)
3. Sparrow Wallet (HWI + Electrum, public/local/remote nodes)

## Verification Commands
```
gettxoutsetinfo
getblockchaininfo
gettransaction <txid>
```
